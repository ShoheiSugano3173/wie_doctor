\section{Webアプリの検証実験}\label{chap:3.6}
開発したWebアプリ機能の有用性を確認するため，2018年5月に検証実験を行った．

2週間，機器を導入して排泄情報を収集しそのデータを基に，排泄パターン表を作成した．その後2週間，自動作成された排泄パターン表の時刻にあわせて排泄介護を行い，排泄パターン表の有効性を検証した．


\subsection{検証実験方法}\label{chap:3.6.1}
Webアプリにおける排泄パターン表生成機能の試験は以下のように行った．

\begin{itemize}
\item 期間\\
2018/5/15〜2018/6/14\\
(途中，機器のエラーがあり，実働は18日間)
\item 被験者数\\
6名（男性0名，女性6名）
\item 実験場所\\
都内介護施設
\item 収集したデータ数（データの定義）\\
94件（製品が検知したもの）
\item 実験方法
    \begin{enumerate}
    \item これまで施設が行っていた定時おむつ交換を通常通り行ってもらう．おむつ交換後に，排泄結果をベッドサイドの入力スイッチから，もしくはWebアプリから入力してもらう．
    \item 機器導入から2週間後，排泄結果を基に，定時交換のタイミングの調整など含めた排泄ケアプランを見直し．
    \item 見直し後，新たな定時交換タイミングやおむつ・パッドサイズで排泄ケアを実施．
    \end{enumerate}
\end{itemize}

前半の2週間で機器を導入して排泄情報を収集し，そのデータを基に排泄パターン表を作成した．その後2週間，自動作成された排泄パターン表の時刻にあわせて排泄介護を行い，排泄パターン表の有効性を検証した．

\subsection{検証実験結果}\label{chap:3.6.2}
実験の結果として，管理表の排泄予測時刻と実際の介護時に確認した排泄の有無の整合性は約72\%であった．今後収集するデータ量がさらに長期間になることで，自動作成時の精度は高まっていくため，このアプリ開発の目的である「要介護者一人ひとりの異なる排泄リズムに対応する」ことは達成できると考えられる．

\subsection{排泄パターン表の効果}\label{chap:3.6.3}
臨床現場で排泄パターン表を使用した際の効果について述べる．以下は実際の排泄パターン表である．横軸が1日24時間を表し，縦軸が各時間帯で何回排泄したかを表している．例えば朝方6時はこれまで3回排泄されていたと，グラフから読み取れる． またグラフに入れられた縦点線は，この施設の現在の定時おむつ交換の予定時間帯である．グラフの通り，この要介護者の方は20時台，23時台，深夜2時の3箇所で排泄が多く見られるが，定時おむつ交換の予定時間帯は，20時前，深夜1時，明け方5時の3箇所であり，排泄時刻と大きくずれている．そのため，この要介護者に対する定時おむつ交換では，空振りや尿便漏れが発生し，ゆえに要介護者の負担を増加させている可能性がある．そこで本機能を用い最適な定時おむつ交換タイミングを算出し，適切な定時おむつ交換タイミングを提案した．これにより，P-mSHELLモデルを用い定義したインシデントである，おむつ交換の空振りや，尿便漏れを防ぐ可能性があることがいえる．よって，本製品は，おむつ交換業務における負担を軽減し，要介護者および介護者の身体的精神的負担を軽減させる可能性があるといえる．
